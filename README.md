# Дневник тренировок — веб-версия

Это тот же трекер тренировок, что был в виде Claude-артефакта, но теперь
как обычный сайт. Работает на любом устройстве и в любом браузере,
данные хранятся в облачной базе (Supabase), а не в браузере.

## Что внутри

- `src/App.jsx` — всё приложение (тренировки, замеры, прогресс, профиль)
- `src/supabaseClient.js` — подключение к базе данных
- `supabase/schema.sql` — SQL для создания таблиц в Supabase
- PIN-защита — простой пароль на вход, чтобы случайные люди со ссылкой
  не попали в твои данные

## Шаг 1. Создай проект в Supabase (база данных)

1. Зайди на [supabase.com](https://supabase.com) → Sign up (можно через GitHub) → New project.
2. Дай проекту любое имя, придумай пароль базы данных (сохрани его куда-то — он отдельный от PIN-кода сайта), выбери регион (любой европейский — ближе к тебе).
3. Подожди 1-2 минуты, пока проект создастся.
4. Слева в меню — **SQL Editor** → **New query**.
5. Открой файл `supabase/schema.sql` из этого проекта, скопируй весь текст, вставь в редактор и нажми **Run**.
6. Должно появиться "Success. No rows returned" — значит таблицы созданы.
7. Слева в меню — **Project Settings** (значок шестерёнки) → **API**.
   Тебе нужны два значения:
   - **Project URL** (выглядит как `https://xxxxx.supabase.co`)           https://tscognyzsxebvcfafiud.supabase.co/rest/v1/
   - **anon public** ключ (длинная строка в разделе "Project API keys")      sb_publishable_eN2hQR1fkuCwWtpA9-IgjA_wBNffn7y

Сохрани эти два значения — они понадобятся на шаге 3.

## Шаг 2. Залей код на GitHub

1. Зайди на [github.com](https://github.com), создай новый репозиторий (например `workout-tracker`).
2. Залей туда все файлы из этой папки (`workout-tracker-web`). Проще всего:
   - Скачай папку себе на компьютер
   - На странице нового репозитория на GitHub нажми "uploading an existing file" и перетащи все файлы и папки
   - Либо, если знаешь git: `git init`, `git add .`, `git commit -m "init"`, дальше по инструкции с GitHub

   **Важно:** файл `.env` не должен попасть в репозиторий (он и так в `.gitignore`, но на всякий случай проверь, что не загрузил его вручную) — там в итоге будут чувствительные ключи, которые лучше держать только в Vercel.

## Шаг 3. Задеплой на Vercel (сайт)

1. Зайди на [vercel.com](https://vercel.com) → Sign up через GitHub.
2. **Add New** → **Project** → выбери свой репозиторий `workout-tracker`.
3. Vercel сам определит, что это Vite-проект. Перед тем как нажать Deploy, разверни **Environment Variables** и добавь:
   - `VITE_SUPABASE_URL` = твой Project URL из Supabase
   - `VITE_SUPABASE_ANON_KEY` = твой anon public ключ из Supabase
4. Нажми **Deploy**. Через минуту-две получишь ссылку вида `https://workout-tracker-xxxx.vercel.app`.

## Шаг 4. Первый запуск

1. Открой ссылку на телефоне или компьютере.
2. Приложение предложит создать PIN-код — введи любой (минимум 4 символа), повтори, нажми "Создать и войти".
3. Дальше при каждом новом открытии сайта (новая вкладка/перезапуск браузера) будет просить этот PIN.
4. Готово — дальше всё работает как раньше: тренировки, вес, прогресс, профиль.

## Как обновлять сайт после правок кода

Просто заливаешь новую версию файлов в тот же GitHub-репозиторий (заменяешь файлы) —
Vercel автоматически пересоберёт и обновит сайт за минуту-две.

## Важные отличия от версии в Claude-артефакте

- **Хранение данных**: раньше всё лежало в `window.storage` (работало только
  в самом чате Claude и было привязано к одному устройству в Safari).
  Теперь данные в Supabase — доступны с любого устройства и браузера
  по одной ссылке.
- **Экспорт/импорт в Профиле** оставлен как был — можно делать резервную
  копию текстом на случай, если когда-то понадобится перенести данные
  в другое место.
- **PIN-защита** — новое, чтобы случайный человек со ссылкой не увидел
  и не поменял твои данные. Это не банковский уровень защиты (ключ
  Supabase технически виден в коде сайта — так работает Supabase для
  фронтенд-приложений), но для личного трекера это нормально.

## Если что-то не работает

Самые частые проблемы:
- **Белый экран** — скорее всего не указаны или неверны `VITE_SUPABASE_URL`/`VITE_SUPABASE_ANON_KEY` в Vercel → Settings → Environment Variables.
- **"Не удалось загрузить данные"** — проверь, что SQL-скрипт из `supabase/schema.sql` выполнился без ошибок в Supabase.
- **Забыл PIN** — зайди в Supabase → Table Editor → `app_auth`, удали строку с `id = 1`, обнови сайт — он попросит создать PIN заново.
