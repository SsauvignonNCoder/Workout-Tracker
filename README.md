# Дневник тренировок — веб-версия (несколько пользователей)

Сайт, где каждый человек заводит свой собственный аккаунт и видит только
свои тренировки, вес и прогресс. Вход — либо через Telegram (если открыто
через бота, автоматически, без пароля), либо через имя + PIN (в обычном
браузере).

## Что внутри

- `src/App.jsx` — всё приложение (тренировки, замеры, прогресс, профиль)
- `src/supabaseClient.js` — подключение к базе данных
- `api/telegram-auth.js` — серверная проверка входа через Telegram
- `api/browser-auth.js` — серверная регистрация/вход по имени + PIN
- `supabase/schema.sql` — SQL для создания таблиц (для **новой** установки)
- `supabase/migration-to-multiuser.sql` — SQL для **обновления** уже
  существующей старой (одиночной) версии до многопользовательской

## Если ты уже разворачивал старую (одиночную) версию

У тебя уже есть проект в Supabase и Vercel с одной общей записью на всех.
Нужно один раз выполнить миграцию, которая создаст таблицу пользователей
и перенесёт твои существующие тренировки в неё, не теряя данных.

1. Зайди в Supabase → **SQL Editor** → **New query**.
2. Открой файл `supabase/migration-to-multiuser.sql`, скопируй весь текст,
   вставь и нажми **Run**. Скрипт сам создаст пользователя "Никита" из
   твоих старых данных и перенесёт туда все тренировки/замеры/профиль.
3. Если до этого был привязан Telegram (через старую `app_auth`) — он
   автоматически перенесётся, дальше всё работает как раньше для тебя.
4. Залей обновлённые файлы из этого проекта на GitHub (как минимум
   `src/App.jsx`, `index.html`, всю папку `api/`).
5. В Vercel добавь новую переменную окружения **`SUPABASE_SERVICE_ROLE_KEY`**
   (см. ниже, где её взять) и сделай **Redeploy**.
6. Готово — теперь любой человек может зайти на твой сайт/бота и создать
   свой собственный аккаунт.

## Если разворачиваешь с нуля (новый проект)

### Шаг 1. Supabase

1. [supabase.com](https://supabase.com) → New project → дождись создания.
2. **SQL Editor** → **New query** → вставь содержимое `supabase/schema.sql` → **Run**.
3. **Project Settings** (шестерёнка) → **API Keys** — тебе нужны:
   - **Project URL** (`https://xxxxx.supabase.co`)
   - **Publishable key** (это и есть anon-ключ для фронтенда)
   - **Secret key** (он же `service_role` — для серверных функций, **никогда** не для кода фронтенда)

### Шаг 2. GitHub

Залей все файлы этого проекта в свой репозиторий (через drag-and-drop на
странице "uploading an existing file" или через git). Не загружай файл
`.env` — там будут чувствительные ключи, которые нужны только в Vercel.

### Шаг 3. Vercel

1. [vercel.com](https://vercel.com) → **Add New → Project** → выбери свой репозиторий.
2. **Environment Variables** — добавь все четыре:
   - `VITE_SUPABASE_URL` = твой Project URL
   - `VITE_SUPABASE_ANON_KEY` = твой Publishable key
   - `SUPABASE_SERVICE_ROLE_KEY` = твой Secret key
   - `TELEGRAM_BOT_TOKEN` = токен бота (см. шаг 4)
3. **Deploy**.
4. После деплоя зайди в **Settings → Domains** и используй именно
   **стабильный** домен (без случайных букв в адресе типа `-4o424je2v-`)
   — это важно для следующего шага с Telegram.

### Шаг 4. Telegram-бот

1. В Telegram напиши [@BotFather](https://t.me/BotFather) → `/newbot` →
   дай имя и username (заканчивается на `bot`) → получишь **токен**.
2. Вставь токен в Vercel как `TELEGRAM_BOT_TOKEN` (см. шаг 3) → Redeploy.
3. Снова @BotFather → `/mybots` → твой бот → **Bot Settings** →
   **Menu Button** → **Configure menu button** → укажи стабильный URL
   сайта из Vercel и название кнопки (например "Открыть дневник").

## Как теперь работает вход

- **Через Telegram-бота** — открываешь бота, нажимаешь кнопку меню,
  приложение видит твой Telegram-аккаунт и **автоматически** создаёт
  тебе персональный аккаунт при первом заходе. Без пароля.
- **Через обычный браузер** (ссылка на сайт напрямую) — на экране входа
  нажимаешь "Нет аккаунта? Создать новый", придумываешь имя и PIN
  (минимум 4 символа). В следующий раз входишь тем же именем и PIN.
- Каждый аккаунт (хоть через Telegram, хоть через браузер) видит **только
  свои** тренировки, вес и профиль — данные полностью разделены.
- Кнопка выхода — иконка рядом с переключателем темы в шапке приложения.

## Как обновлять сайт после правок кода

Заливаешь новую версию изменённых файлов в тот же GitHub-репозиторий —
Vercel автоматически пересоберёт и обновит сайт за минуту-две.

## О безопасности

Это не полноценная система авторизации с токенами (как настоящий Supabase
Auth), а упрощённая схема для личного использования среди доверенных
людей:
- PIN хранится как SHA-256 хэш, не в открытом виде.
- Telegram-вход проверяется криптографической подписью на сервере —
  это надёжно, подделать нельзя.
- Но: разделение данных по пользователям обеспечивается логикой
  приложения, а не строгими правами доступа на уровне базы (Row Level
  Security открыт всем, кто знает анонимный ключ — а он есть в коде
  сайта по дизайну Supabase). Кто-то технически продвинутый теоретически
  мог бы обратиться к базе напрямую. Для трекера тренировок среди близких
  людей это разумный компромисс, но не подходит для чувствительных данных.

## Если что-то не работает

- **Белый экран** — проверь `VITE_SUPABASE_URL` / `VITE_SUPABASE_ANON_KEY` в Vercel.
- **"Сервер не настроен"** при входе — не хватает `SUPABASE_SERVICE_ROLE_KEY` или `TELEGRAM_BOT_TOKEN` в Vercel → добавь → Redeploy.
- **"Не удалось загрузить данные"** — проверь, что SQL из `supabase/schema.sql` (или миграция) выполнился без ошибок.
- **В Telegram не входит автоматически** — проверь, что URL в Menu Button в BotFather совпадает со стабильным production-доменом в Vercel (без случайных символов в адресе).
- **"Это имя уже занято"** при регистрации в браузере — выбери другое имя или войди с тем PIN, что уже был задан для этого имени.
